+ encountered non-expected behavior with `stringr::str_glue`:
	+ per default, it would replace `NA`'s in a substring element with character NA's; e.g. `stringr::str_glue("{ 'GW03.3' }:PS{ '2' }{ NA }")` -> `GW03.3:PS2NA`
	+ this will not trigger any downstream `NA` checks; e.g. `is.na(stringr::str_glue("{ 'GW03.3' }:PS{ '2' }{ NA }")) == FALSE`; in consequence, missing data might go unseen
	+ better use `.na = NULL`, e.g. `stringr::str_glue("{ 'GW03.3' }:PS{ '2' }{ NA }", .na = NULL)` -> `NA`
